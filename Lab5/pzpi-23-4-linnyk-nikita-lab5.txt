3 ОПИС ВИКОНАНОЇ РОБОТИ
Назва інструменту та версія: Anthropic Claude Sonnet 4.6, Claude-Design
Мета використання: проєктування та вдосконалення користувацького інтерфейсу веб- і мобільного клієнтів системи «LangBang», розробка дизайн-системи, структури навігації та компонентів адміністративної панелі, підтримки локалізації й інтернаціоналізації, а також допомога у формуванні архітектури front-end частини системи та UX-рішень для освітньої платформи.
Внесок автора: самостійне проєктування архітектури front-end частини системи, реалізація SPA веб-клієнта на React / TypeScript і мобільного клієнта на React Native / Expo, створення окремих користувацьких та адміністративних інтерфейсів, реалізація функціональної логіки взаємодії з REST API серверної частини, впровадження підтримки української та англійської мов, адаптація інтерфейсу відповідно до регіональних налаштувань і тем оформлення, а також інтеграція та доопрацювання рішень, запропонованих інструментом, відповідно до вимог проєкту.
3.1 Короткий опис проєкту 
        Назва системи: «LangBang — програмна система для вивчення іноземних мов». 
LangBang являє собою інтерактивну освітню платформу, яка поєднує класичне навчання іноземним мовам з технологіями штучного інтелекту, персоналізованим навчальним маршрутом і соціальною взаємодією користувачів. Платформа допомагає вдосконалювати знання іноземних мов завдяки інтелектуальному підбору навчальних матеріалів, спілкуванню з іншими користувачами відповідного рівня володіння мовою і тренуванню зі ШІ-помічником.        
Система реалізована як розподілений програмний продукт, що складається із серверної частини на Node.js / Express з базою даних PostgreSQL, веб-клієнта на React (Vite) і мобільного клієнта на React Native / Expo. Усі клієнти взаємодіють із сервером через єдиний REST API з Bearer JWT-автентифікацією.
3.2 Заплановані функції
Серверна частина: REST API на Node.js / Express; PostgreSQL для збереження користувачів, прогресу й контенту; WebSocket-сервер для чату в реальному часі; JWT + OAuth 2.0; S3-сумісне сховище для медіа; push-сповіщення; шлюз інтеграції зі ШІ-сервісами; 
Веб-клієнт: SPA на React / TypeScript; реєстрація і вхідне тестування рівня CEFR; навчальні модулі; чат зі співрозмовниками; тренування зі ШІ; адмін-панель; підтримка темної / світлої теми та локалізації; доступ до мікрофона для вправ на вимову.
Мобільний клієнт: React Native для Android та iOS; синхронізація з вебверсією; push-сповіщення; обмежений офлайн-доступ; камера і мікрофон для голосових вправ.
Інтеграція ШІ (наступні релізи): адаптивний навчальний маршрут; розпізнавання мовлення; генерація вправ; ШІ-співрозмовник; автоматичний переклад; модерація контенту; підбір співрозмовників.
3.3 Реалізовані функції
Серверна частина:
– REST API на Node.js / Express 5 з модульною трирівневою архітектурою (routes → controllers → services); 
– PostgreSQL з Prisma ORM, повна схема даних з ER-моделлю (USER, COURSE, LESSON, EXERCISE, ENROLLMENT, EXERCISE_PROGRESS, MESSAGE та ін.); 
– JWT-автентифікація з access-токенами в памʼяті, refresh-токенами в httpOnly cookie і tokenVersion для анулювання сесій; 
– ролевий контроль доступу (admin, contentManager, moderator, user);
– endpoint /health для readinessProbe та livenessProbe; 
– horizontal scaling через Kubernetes Deployment + Service з демонстрацією на 1, 2 та 4 репліках.
Веб-клієнт: 
– SPA на React 19 + Vite + TypeScript; 
– TanStack Query v5 для серверного стану, Zustand для аутентифікації; 
– Axios з interceptor для автоматичного оновлення токенів; 
– дизайн-система на CSS Custom Properties з темною і світлою темою; 
– інтернаціоналізація (українська / англійська) з Intl API для дат, чисел і часових поясів; 
– окремі інтерфейси для звичайних користувачів і адміністраторів; 
– адмін-функціонал: управління користувачами, експорт / імпорт даних, резервні копії; 
– захист від самодеградації прав адміністратора.
Мобільний клієнт: 
– React Native / Expo з файловою маршрутизацією через Expo Router; 
– чотиришарова архітектура (навігація, дані, мережа, представлення); 
– Expo SecureStore для refresh-токена; React Query + Zustand; 
– функціональність: курси, флеш-картки, тести, AI-діалог, прогрес.
Навантажувальне тестування: проведено через Locust для 1, 2 і 4 реплік API-сервера; підтверджено майже лінійний приріст RPS при масштабуванні з 1 до 2 реплік і виявлено, що при 4 репліках вузьким місцем стає PostgreSQL.
3.4 Заплановані, але не виконані функції
– WebSocket-чат у реальному часі — реалізовано лише REST-обмін повідомленнями через таблицю Message без push-каналу; 
– Розпізнавання мовлення та аналіз вимови — потребує власної ML-інфраструктури або платних API і не входить у скоуп MVP;
 – OAuth 2.0 через сторонні сервіси — реалізовано лише класичну реєстрацію через email + пароль; 
– S3-сумісне сховище для медіа і push-сповіщення на мобільні — інфраструктурно складно для навчального проєкту; 
– Повноцінна інтеграція ШІ — перенесено на наступні випуски.
4 ВИСНОВКИ
У результаті роботи протягом двох семестрів реалізовано основну частину запланованої системи: серверну частину, веб-клієнт з адмін-панеллю, мобільний клієнт і продемонстровано можливість горизонтального масштабування. Поза скоупом залишилися переважно інфраструктурно-важкі компоненти і складні ШІ-функції, які потребують окремих ресурсів. Сформована архітектура є модульною і готова до подальшого розширення.
