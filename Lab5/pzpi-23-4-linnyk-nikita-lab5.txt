3 ОПИС ВИКОНАНОЇ РОБОТИ
Назва інструменту та версія: Anthropic Claude Sonnet 4.6, Claude-Design
Мета використання: проєктування та вдосконалення користувацького інтерфейсу веб- і мобільного клієнтів системи «LangBang», розробка дизайн-системи, структури навігації та компонентів адміністративної панелі, підтримки локалізації й інтернаціоналізації, а також допомога у формуванні архітектури front-end частини системи та UX-рішень для освітньої платформи.
Внесок автора: самостійне проєктування архітектури front-end частини системи, реалізація SPA веб-клієнта на React / TypeScript і мобільного клієнта на React Native / Expo, створення окремих користувацьких та адміністративних інтерфейсів, реалізація функціональної логіки взаємодії з REST API серверної частини, впровадження підтримки української та англійської мов, адаптація інтерфейсу відповідно до регіональних налаштувань і тем оформлення, а також інтеграція та доопрацювання рішень, запропонованих інструментом, відповідно до вимог проєкту.
3.1 Короткий опис проєкту 
        LangBang — це повнофункціональна платформа для вивчення іноземних мов, побудована за принципом мікронавчання. Система складається з трьох незалежних, але інтегрованих компонентів: веб-застосунку на React (Next.js), мобільного застосунку на Flutter / Dart та серверної частини на Node.js / Express 5 з базою даних PostgreSQL і ORM Prisma.
Мобільний клієнт розроблено засобами Flutter SDK та мови програмування Dart. Застосунок компілюється у нативний ARM-код для платформи Android і взаємодіє з сервером виключно через REST API з JWT-авторизацією.
Платформа реалізує повний цикл вивчення мов: реєстрацію та керування профілем користувача, запис на мовні курси, щоденне відпрацювання матеріалу через картки та тести, AI-діалог для розмовної практики та гейміфікацію через таблицю лідерів і систему досягнень.
3.2 Заплановані функції
Серверна частина (Node.js / Express 5):
– REST API з JWT-авторизацією (access + refresh токени);
– управління користувачами, курсами та уроками;
– система прогресу: відстеження пройдених вправ, XP, серій (streaks);
– таблиця лідерів із фільтрацією за часовим діапазоном (день / тиждень / місяць / весь час);
– інтеграція з OpenAI API для генерації відповідей у діалоговому режимі;
– горизонтальне масштабування через Kubernetes з автоматизованим розгортанням.
Мобільний клієнт (Flutter / Dart):
– компіляція у нативний Android-застосунок;
– JWT-авторизація з автоматичним оновленням токенів (interceptor у Dio);
– навігація через GoRouter з StatefulShellRoute (збереження стану вкладок);
– управління станом через flutter_riverpod: AsyncNotifier, Notifier, FutureProvider.autoDispose;
– флеш-картки з 3D-анімацією перевертання (AnimationController + Matrix4.rotationY);
– Quiz: 4 варіанти відповіді, кольорове підсвічування результату, оновлення XP;
– AI-діалог: чат-інтерфейс з polling нових повідомлень кожні 3 секунди;
– таблиця лідерів із подіумом топ-3 та фільтрацією за перiодом;
– редагування профілю, налаштування щоденної цілі, push-сповіщення.
Веб-клієнт:
– адміністрування курсів, уроків і вправ;
– аналітика прогресу студентів.
3.3 Реалізовані функції
Серверна частина:
– реалізовано повноцінний REST API на Node.js з фреймворком Express 5. База даних — PostgreSQL; для схеми та міграцій використовується Prisma ORM. Автентифікація побудована на парі access / refresh JWT-токенів: access-токен живе 15 хвилин, refresh-токен — 7 днів; при кожному запиті до захищених маршрутів middleware перевіряє підпис і термін дії;
– реалізовані REST-маршрути охоплюють: реєстрацію та вхід, оновлення токена (POST /auth/refresh), CRUD курсів та уроків, запис на курс, отримання та відправку вправ з автоматичним нарахуванням XP, таблицю лідерів із SQL-запитами агрегації за обраний часовий діапазон, профіль користувача, налаштування (денна ціль, push-сповіщення), AI-діалог через OpenAI Chat Completions API;
– для автоматизованого розгортання в середовищі Kubernetes реалізовано локальну вебпанель LangBang Lab 4 Setup. Через UI користувач може запустити повне розгортання проєкту, обрати режим бази даних, задати змінні середовища, створити Kubernetes Secret, застосувати ConfigMap, розгорнути PostgreSQL та API-сервер, виконати Prisma migrations і seed, а також встановити початкову кількість реплік. Після розгортання в блоці Operations можна масштабувати API, запустити port-forward, перевірити доступність endpoint-ів, встановити та запустити Locust UI і виконати очищення Kubernetes-ресурсів. PowerShell-скрипт lab4-setup.ps1 залишено як альтернативний консольний спосіб для виконання тих самих дій без вебінтерфейсу.
Веб-клієнт:
– SPA на React 19 + Vite + TypeScript;
– TanStack Query v5 для серверного стану, Zustand для аутентифікації;
– Axios з interceptor для автоматичного оновлення токенів;
– дизайн-система на CSS Custom Properties з темною і світлою темою;
– інтернаціоналізація (українська / англійська) з Intl API для дат, чисел і
часових поясів;
– окремі інтерфейси для звичайних користувачів і адміністраторів;
– адмін-функціонал: управління користувачами, експорт / імпорт даних,
резервні копії;
– захист від самодеградації прав адміністратора
Мобільний клієнт (Flutter / Dart)
Мобільний застосунок реалізовано на Flutter SDK з використанням мови Dart. Проєкт побудовано за feature-first архітектурою: функціональні модулі поділено на шари data/, domain/ та presentation/, що розмежовують роботу з API, бізнес-логіку та інтерфейс користувача.
Основні технічні рішення мобільного клієнта наведено нижче:
– навігацію реалізовано через GoRouter із використанням StatefulShellRoute.indexedStack, що дає змогу зберігати стан основних вкладок. Окремий стек авторизації відповідає за екрани входу, реєстрації та відновлення пароля;
– управління станом побудовано на flutter_riverpod. AuthNotifier відповідає за автентифікацію та відновлення сесії, PracticeNotifier — за стан режиму практики, а FutureProvider.autoDispose — за кешування й оновлення серверних даних;
– мережевий рівень реалізовано через DioClient. Він додає JWT-токен до запитів, обробляє помилки авторизації, оновлює токен і повторює запити після успішного refresh-запиту;
– моделі даних створено з використанням Freezed та json_serializable, що забезпечує незмінність об’єктів, автоматичне оновлення через copyWith і серіалізацію JSON-даних;
– режим практики містить флеш-картки з 3D-анімацією, тестові завдання та AI-діалог із періодичним оновленням повідомлень;
– оформлення застосунку винесено в константи AppColors, AppTextStyles і AppSpacing, а спільні UI-компоненти розміщено в shared/widgets/ для зменшення дублювання коду.
3.4 Заплановані, але не виконані функції
– WebSocket-чат у реальному часі — реалізовано лише REST-обмін повідомленнями через таблицю Message без push-каналу; 
– Розпізнавання мовлення та аналіз вимови — потребує власної ML-інфраструктури або платних API і не входить у скоуп MVP;
 – OAuth 2.0 через сторонні сервіси — реалізовано лише класичну реєстрацію через email + пароль; 
– S3-сумісне сховище для медіа і push-сповіщення на мобільні — інфраструктурно складно для навчального проєкту; 
– Повноцінна інтеграція ШІ — перенесено на наступні випуски.
4 ВИСНОВКИ
У результаті роботи протягом двох семестрів реалізовано основну частину запланованої системи: серверну частину, веб-клієнт з адмін-панеллю, мобільний клієнт і продемонстровано можливість горизонтального масштабування. Поза скоупом залишилися переважно інфраструктурно-важкі компоненти і складні ШІ-функції, які потребують окремих ресурсів. Сформована архітектура є модульною і готова до подальшого розширення.
